## Constitution Structure

One document at this scope — the surface area is narrower than a full application stack.

### [Constitution Core](constitution-core.md) ← **START HERE**

**Contains**:
- Mandatory skill file structure & frontmatter schema
- Runtime variable & repo resolution rules
- Output format rules (ranked lists, NEXT STEPS, ROI formula)
- Communication style conventions
- Skill invocation rules
- Cross-tool compatibility guidance (GHCP vs. Claude Code vs. psis)
- Contributing / git workflow
- Guardrails convention

**When to use**: Before authoring or modifying any skill. Before reviewing a PR to this repo.

---

## Quick Start

1. **Adding a new skill?** Read [Constitution Core](constitution-core.md) Section I (file structure) before creating anything.
2. **Modifying an existing skill?** Check the relevant section — output format changes have their own rules (Section III).
3. **Reviewing a PR?** Use the constitution to cite specific rule violations rather than re-explaining conventions inline.
4. **Unsure about a pattern?** If it contradicts the constitution, it's wrong. If the constitution is silent, propose a rule addition via issue before implementing ad-hoc.
